Background

Why Point Cloud Classification?

⚠ The Core Challenge
  • Point clouds are unordered — no natural grid structure
  • Traditional CNNs designed for ordered grids (images/voxels)
  • Permutation must not change the output
  • Key for robotics, autonomous driving, AR/VR
✓ Our Solution
  • PointNet: per-point MLP + global max-pooling
  • PointNet++: hierarchical local feature grouping
  • Evaluated on ModelNet40 (40-class, standard benchmark)

① Geometric Ambiguity
dresser / nightstand / wardrobe share near-identical boxy geometry. Only fine surface details differentiate them — which point clouds tend to lose.
② Class Imbalance
Under-represented classes (flowerpot: ~39 train, tvstand: ~67 train) have fewer examples, producing weaker discriminative features.
③ Scale Normalization
Unit sphere normalization removes absolute size — a critical cue that humans use to distinguish desk from nightstand or table.
🔧 Proposed Improvements
  • Add normal/edge features as auxiliary input
  • Focal loss to handle class imbalance
  • Fine-tune with domain-specific augmentation

PointNet++ outperforms PointNet by +1.84% (90.06% → 91.90%) via hierarchical local feature learning

Domain Gap
~78% accuracy drop on ScanObjectNN reveals the challenge of synthetic-to-real generalization in 3D vision
Hierarchical local geometry + data augmentation = state-of-the-art reproducibility on ModelNet40 3D shape classification
